## 2. 【最優先】社長指示：SaaSの死×AI代替リスクの定量化

> **社長指示優先度2（2026/04/06）。SaaSの死×AI代替リスクの定量化が最重要**

### 2-1. 「SaaSの死」テーゼとは何か

**SaaSの死（Death of SaaS）テーゼの骨子：**

LLM（大規模言語モデル）とAIエージェントの進化により、従来型SaaSが提供していた「機能モジュール」は、AIが自然言語でオンデマンドに代替する。ユーザーは個別ソフトウェアを「購入・契約」する必要がなくなり、AIオーケストレーション層が複数機能を統合処理する。

**構造的脅威の3層モデル：**

```
【第1層】UI/UX代替
   従来：GUIベースの画面操作
   AI後：自然言語コマンド → LLMが実行
   影響：フロントエンドSaaSの陳腐化

【第2層】ワークフロー代替
   従来：SaaS間をAPIで手動連携
   AI後：AIエージェントが自律判断で連携
   影響：中間統合ツールの消滅

【第3層】意思決定代替
   従来：ダッシュボード→人間が判断
   AI後：AIが分析・提案・実行まで自動化
   影響：分析系SaaSの根本的不要化
```

### 2-2. スマレギへのSaaS死テーゼ適用：リスク層別分析

#### ◆ スマレギが「死にやすい機能」vs「死にくい機能」

| 機能カテゴリ | AI代替リスク | 理由 | 死亡確率（深井推定） |
|------------|------------|------|-------------------|
| **売上分析・レポーティング** | 🔴 極高 | LLMが自然言語で瞬時に代替可能。BI/分析SaaSは最初に消える | 80% |
| **在庫管理・発注最適化** | 🟡 中 | AI予測は優秀だが、実店舗オペレーションとの統合が必要 | 45% |
| **シフト・勤怠管理（タイムカード）** | 🟡 中 | AI最適化は有効。ただし労働法・就業規則の複雑性が参入障壁 | 40% |
| **セルフオーダー（ウェイター）** | 🟢 低 | タブレット/QRコードの物理インターフェースが本質。AI化は機能強化 | 15% |
| **決済処理（ペイメント）** | 🟢 極低 | 金融ライセンス・セキュリティ要件・物理端末。構造的参入障壁 | 5% |
| **POSコア（会計・レシート）** | 🟢 低 | 税務・法令遵守の物理的執行が必要。法的責任の所在が参入障壁 | 20% |

#### ◆ 「物理的接点」が堀になるか？—深井の評定

**結論：物理的接点は堀の必要条件だが十分条件ではない**

```
スマレギの堀の構造：

【強い堀】
  ✅ ハードウェア連携（レシートプリンタ/バーコードリーダー/キャッシュドロア）
     → AI単独では代替不可能。物理機器との認証・通信プロトコルが参入障壁
  ✅ 決済ライセンス・PCI DSS準拠
     → 新規AI参入者にとって2〜3年・数億円のコスト
  ✅ 顧客データの粘着性（販売履歴/顧客データ/在庫マスタ）
     → スイッチングコスト：中小企業で推定¥50〜200万/店舗
  ✅ 税務・法令コンプライアンス（インボイス/電帳法）
     → 法改正への継続対応が必要 = 常に更新コストが発生 = チャーン抑制

【弱い/脆弱な堀】
  ⚠️ 分析・レポーティング機能 → LLMで即代替可能
  ⚠️ マニュアル的ワークフロー → AIエージェントが浸食
  ⚠️ 中小企業の価格感応度 → 「無料AIツール」登場時の離脱リスク
```

### 2-4. 「40%ルール」41.5%の持続性検証

**Rule of 40（売上成長率 + 営業利益率 ≥ 40%）**

スマレギは直近で41.5%を達成。優良SaaS企業の基準（40%）をクリア。

**しかし深井は以下の構造的問題を指摘する：**

```
41.5%の内訳推定：
  売上成長率：約+25〜30%
  営業利益率：約+11.5〜16.5%
  （両者のバランスは成長投資の継続を示す）

持続性のリスク要因：
  1. AI投資コスト増大
     → 競合に対抗するためのAI機能開発費が急増
     → 研究開発費/売上比が5%→10%へ上昇すれば営業利益率▲5%
     → Rule of 40が40%を下回るリスク：中程度（★★☆）

  2. 成長率鈍化の数学的必然
     → 市場浸透率が上がるほど追加成長は難しくなる
     → 国内SMB市場の飽和（推定50〜60%がまだ未デジタル化）
     → 逆に言えばまだ追い風は残存

  3. ARPU圧迫（競争激化）
     → 競合の無料化・低価格化戦略
     → AI代替による機能価値の希薄化
```

**深井評定：41.5%は2026-2027年は維持可能。2028年以降はシナリオB依存で37〜42%のレンジに分岐。**

## 4. DCFバリュエーション

### 4-1. 前提条件（全明記）

#### ① マクロ前提

| パラメータ | 値 | 根拠 |
|----------|-----|------|
| リスクフリーレート（Rf） | **2.380%** | JGB10Y（社長確認値。2026-04-06） |
| エクイティリスクプレミアム（ERP） | **6.5%** | 日本市場標準（Damodaran 2026推定値参照） |
| Beta（β） | **1.25** | グロース系SaaS株。東証グロース市場βの中位推定 |
| 株主資本コスト（Re） | **2.380% + 1.25 × 6.5% = 10.505%** | CAPM |
| 負債コスト（Rd） | **0.8%** | 低有利子負債。銀行借入金利推定 |
| 実効税率（Tc） | **30%** | 日本法人税率（地方税含む） |
| 資本構成：E/(D+E) | **90%** | 自己資本比率推定 |
| 資本構成：D/(D+E) | **10%** | 有利子負債比率推定 |
| **WACC** | **Re×90% + Rd×(1-30%)×10%** | **= 10.505%×0.9 + 0.8%×0.7×0.1** |
| | **≈ 9.511%** | 計算値 |
| ターミナル成長率（g） | **2.5%** | 長期インフレ+実質GDP。名目GDP成長率参考 |

**WACC計算詳細：**
```
Re = 2.380% + 1.25 × 6.5% = 2.380% + 8.125% = 10.505%
WACC = 10.505% × 0.90 + 0.8% × (1-0.30) × 0.10
     = 9.455% + 0.056%
     = 9.511% ≈ 9.51%
```

#### ② 売上・FCF成長前提（基本シナリオ）

| 年度 | 売上成長率 | FCFマージン | FCF（推定） |
|-----|----------|-----------|-----------|

> 売上起点：FY2025実績売上高を¥50億と推定（ARR¥52億前後に整合）

#### ③ DCFキャッシュフロー計算（基本シナリオ）

```
WACC = 9.51%

FY2026: FCF ¥7.5億 → PV = 7.5 / (1.0951)^1 = ¥6.85億
FY2027: FCF ¥9.8億 → PV = 9.8 / (1.0951)^2 = ¥8.17億
FY2028: FCF ¥12.5億 → PV = 12.5 / (1.0951)^3 = ¥9.50億
FY2029: FCF ¥15.0億 → PV = 15.0 / (1.0951)^4 = ¥10.40億
FY2030: FCF ¥17.5億 → PV = 17.5 / (1.0951)^5 = ¥11.07億

PV合計（5年間）= ¥45.99億

ターミナルバリュー（TV）：
  TV = FCF_2030 × (1+g) / (WACC - g)
     = ¥17.5億 × 1.025 / (0.0951 - 0.025)
     = ¥17.9375億 / 0.0701
     = ¥255.9億

TV現在価値 = ¥255.9億 / (1.0951)^5 = ¥161.9億

企業価値（EV）= ¥45.99億 + ¥161.9億 = ¥207.9億

ネットキャッシュ調整（推定）：
  現預金 - 有利子負債 ≈ +¥10〜15億（SaaS企業。キャッシュリッチ）
  → 調整後エクイティ価値 = ¥207.9億 + ¥12億 = ¥219.9億

発行株数（推定）：¥219.9億 / ¥2,237 ≈ 983万株
  → 逆算的に理論株価 = ¥219.9億 ÷ 約980万株 ≈ ¥2,244
```
