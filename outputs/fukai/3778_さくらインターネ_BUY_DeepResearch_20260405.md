### 1-3. 競合優位性の詳細分析

#### ① 石狩データセンター（北海道）— 自然冷却コスト優位

石狩DCは「外気冷却（フリークーリング）」を採用しており、北海道の寒冷な外気を活用してPUE（Power Usage Effectiveness）を業界平均の1.4〜1.6から**1.2前後**まで低減できる。GPU演算はCPUより発熱量が格段に大きいため、この冷却コスト優位は**GPU時代に逆算して正しい設計**であったといえる。

| 指標 | 業界平均 | さくら石狩DC |
|------|---------|-------------|
| PUE | 1.4〜1.6 | 〜1.2 |
| 電力コスト | 高 | 低（外気冷却効果） |
| 冷却CAPEX | 要大型冷却設備 | 最小化 |
| カーボンニュートラル適合 | 課題あり | 相対優位 |

#### ② 国産クラウドというブランド価値（デジタル主権文脈）

日本政府の「デジタル主権」政策は、外資クラウド（AWS/Azure/GCP）への依存リスクを認識し始めている。経済安全保障推進法の枠組みにおいて、**国産クラウド事業者への政策的後押し**が顕在化しており、さくらはこの流れの直接的受益者に位置する。

#### ③ 政府クラウド認定（ガバメントクラウド）

2023年〜2024年にかけ、さくらインターネットは**デジタル庁のガバメントクラウド候補として認定**を受けた。これは国産クラウド史上、最大のカタリストの一つ。AWS・Azure・GCPが独占していた政府調達市場に初めて国産事業者として参入する基盤が整いつつある。

#### ④ 高火力コンピューティング（GPU鯖）

NVIDIA H100/A100クラスのGPUクラスタを提供する「高火力コンピューティング」サービスは、**国内AI企業・大学・研究機関のLLM学習インフラ**として需要急拡大中。NEDO（新エネルギー・産業技術総合開発機構）やNII（国立情報学研究所）等との連携も進む。

### 2-2. 政府クラウド認定の持続性精査（最重要）

#### A. 認定の現状と構造

```
ガバメントクラウド認定プロセス（デジタル庁）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Layer 1: 技術要件審査（セキュリティ/可用性/コンプライアンス）
Layer 2: 運用要件審査（SLA/サポート体制/BCP）
Layer 3: 価格競争力審査（政府調達ルール準拠）
Layer 4: 継続的モニタリング（認定後も定期審査）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
さくら現在地: Layer 3〜4（認定取得済、稼働開始フェーズ）
```

#### B. 持続性を支持する要因（ポジティブ）

**①政策的バックストップの強固さ**

デジタル庁・経産省・内閣府の3省連携で「国産クラウド育成」が明記されており、**少なくとも2030年までの政策的支援継続**が見込まれる。岸田政権→石破政権→次期政権でも「経済安保」は超党派的コンセンサスであり、政策リバーサルリスクは低い。

**②スイッチングコストの高さ**

一度ガバメントクラウドに各省庁のシステムが移行し始めると、**技術的・組織的スイッチングコスト**が極めて高くなる。AWS等への移行は政策的にも困難。さくらの地位は一定期間保護される。

**③ISO27001/ISMAP認証の壁**

政府クラウドに求められるISMAP（情報システムのためのセキュリティ評価制度）認証は取得に相当のコストと時間がかかる。**新規参入障壁として機能**しており、さくらの先行者優位が持続する。

#### C. 持続性を脅かす要因（リスク）

**①技術的追い付きリスク**

```
リスクシナリオ: AWS/Azure等がローカルゾーン戦略強化
→ 日本国内リージョン増設 + 政府向けセキュリティ要件充足
→ 「国産」優位が価格競争力で侵食される
確率評価: 中（30〜40%）
時間軸: 3〜5年
```

**②予算制約・政府調達の不透明性**

日本の財政悪化が続く場合、デジタル化投資予算が削減される可能性がある。また政府調達特有の**競争入札リスク**（毎回の入札更新でシェア低下の可能性）も存在する。

**③技術能力格差の顕在化リスク**

さくらの技術スタックは堅実だが、**フルマネージドサービスの幅広さ**や**グローバル標準のAPI互換性**においてAWSとのギャップがある。政府システムの高度化に伴い、技術要件が厳格化すれば対応コストが増大する。

**④認定更新審査リスク**

ガバメントクラウド認定は**継続的なモニタリングと定期更新審査**がある。大規模障害・セキュリティインシデントが発生した場合、認定取消しのリスクが生じる。

## 4. DCFバリュエーション

### 4-1. WACC算出

#### Step 1: リスクフリーレート（Rf）

```
Rf = 2.380%（日本10年国債 JGB10Y）
取得日時: 2026-04-05 17:31 JST（社長確認値）
参照元: https://jp.investing.com/rates-bonds/japan-10-year-bond-yield-streaming-chart
```

#### Step 2: ベータ（β）

```
β推定根拠:
- 日本クラウド/IT中小型株の業界βレンジ: 1.2〜1.8
- さくらはGPU/政府クラウドのテーマ性が高くボラティリティ大
- 直近1年の株価ボラティリティ（実績）: 高い（テーマ株として急騰・急落あり）

採用β = 1.55（レバードβ）
保守的レンジ: 1.35〜1.75
```

#### Step 3: エクイティリスクプレミアム（ERP）

```
ERP採用値 = 5.5%
根拠:
- Damodaran（2025年推計）日本市場ERP: 約5.0〜6.0%
- 日本株の長期ERP実証値: 約5〜6%
- スモールキャップ追加プレミアム（Size Premium）: +0.5〜1.0%
  ※ 本DCFでは基本ERPに内包してβで対応
採用ERP = 5.5%（保守的中央値）
```

#### Step 4: 株主資本コスト（Re）

```
Re = Rf + β × ERP
   = 2.380% + 1.55 × 5.5%
   = 2.380% + 8.525%
   = 10.905% ≈ 10.9%
```

#### Step 5: 負債コスト（Rd）

```
Rd = 1.5%（推定：日本の銀行借入金利水準、
          さくら規模の企業の短〜中期借入スプレッド込み）
実効税率 Tc = 30.0%（日本法人実効税率）
税引後 Rd = 1.5% × (1 - 0.30) = 1.05%
```

#### Step 6: 資本構成

```
時価ベース資本構成（推定）:
E（株式時価総額）:  約2,700億円  → E/(D+E) = 約93%
D（有利子負債）:   約180億円   → D/(D+E) = 約7%

※ さくらは株式市場での評価が高く、時価ベースでは株式比率が圧倒的
```

#### Step 7: WACC算出

```
WACC = [E/(D+E) × Re] + [D/(D+E) × Rd × (1-Tc)]
     = [0.93 × 10.905%] + [0.07 × 1.05%]
     = 10.141% + 0.074%
     = 10.215% ≈ 10.2%

採用WACC = 10.2%（ベースケース）
保守的レンジ: 9.5% 〜 11.5%
```

### 4-2. DCFモデル前提条件

| 前提項目 | 数値 | 根拠 |
|---------|------|------|
| 予測期間 | 10年（FY2026〜FY2035） | 標準的DCF期間 |
| ターミナル成長率（g） | 2.0% | 日本GDP長期成長率+インフレ考慮 |
| WACC | 10.2% | 上記算出値 |
| Rf | 2.380% | JGB10Y 社長確認値 |
