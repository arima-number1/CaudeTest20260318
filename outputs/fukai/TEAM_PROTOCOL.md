## 第1条：ファイナンス・レーダー（常時起動・解除禁止）

### 1-1. 希薄化リスクラベル（全レポート必須）

全DeepResearchレポートの冒頭に以下のラベルを必ず貼れ。

```
【希薄化リスク：低】… エクイティ発動確率 15%未満
【希薄化リスク：中】… エクイティ発動確率 15〜40%
【希薄化リスク：高】… エクイティ発動確率 40%超、またはMSワラント確認済み
```

**判定基準：**

| 条件 | ラベル |
|------|--------|
| PBR > 1.5倍 + 潤沢CF + 借入余力あり | 低 |
| PBR 1.0〜1.5倍 + CF普通 | 中 |
| PBR < 1.0倍での増資検討、MSワラント存在、赤字続き | 高（警戒） |
| PBR < 1.0倍 + 現金枯渇 + 大型M&A計画 | 高（即時FV下方修正） |

### 1-2. ファイナンス・レーダーの5問チェック

```
Q1: 直近3期のフリーキャッシュフローはプラスか？
Q2: 現在の自己資本比率は業種標準を上回っているか？
Q3: PBR水準でのエクイティ発動は経営的に合理的か？
    （PBR < 1.0倍での増資 = 簿価割り引き売却 = 株主価値破壊）
Q4: 直近の配当・自社株買い方針に変化はないか？
Q5: 大量保有報告書・適時開示に増資示唆の記載はないか？

→ Q1〜Q5が全て「問題なし」: 希薄化リスク【低】
→ 1〜2項目に懸念: 希薄化リスク【中】
→ 3項目以上に懸念 or MSワラント確認: 希薄化リスク【高】
```

## 第2条：キャッシュポジション監視（自動警戒ルール）

### 2-1. ファイナンス警戒アラート発動条件

```
【アラート発動条件】
  買収代金の推定額 > 手元現預金 × 50%

【発動時の対応】
  STEP 1: 「ファイナンス警戒⚠️」ラベルをレポートに付与
  STEP 2: 資金調達方法を3シナリオ（デット/エクイティ/混合）で試算
  STEP 3: 各シナリオの希薄化後EPS × 適正PER = 調整後FVを算出
  STEP 4: 最悪シナリオ（フル希薄化）のFVも必ず提示する

【計算テンプレート】
  現金・預金残高（B/Sより）: A円
  推定買収代金: B円
  デット必要額: B - A（Aで足りない分）
  エクイティ必要額: デットも限界の場合
  警戒アラート条件: B > A × 50%
```

### 2-2. 時価総額別の警戒感度

```
時価総額規模   | 買収可能な「通常範囲」  | アラート閾値
─────────────────────────────────────────────
〜100億円      | 〜10億円           | 5億円超
100〜500億円   | 〜50億円           | 25億円超
500〜2,000億円 | 〜200億円          | 100億円超
2,000億円超    | 〜500億円          | 250億円超

→ これを超える買収は「必ずファイナンス警戒アラートを発動」
```

## 第4条：アウトプット即時同期ルール

```
【深井の鉄則】
  分析が完了した瞬間、mdファイルへ書き込む。
  「調査中」は報告ではない。「出力」こそが仕事だ。

【ファイル更新プロトコル】
  1. DeepResearch完了 → 即座にMDファイル作成・更新
  2. MDファイル完成 → 即座にWordファイル生成
  3. 新銘柄追加 → 即座にdeep_research_index.md / .csv を更新
  4. 判定変更 → 旧ファイルに「v2.0（修正版）」として保存し原版を残す

【ビッグボスがファイルをリロードした瞬間に最新結論が見えている状態を常に維持する】
```

## 第5条：各エージェントの専門責務（確定版）

| エージェント | 責務 | ファイナンス文脈での役割 |
|------------|------|----------------------|
| **浅井（Fact）** | 事実の最速抽出 | 増資・MSワラントのIR開示を最速検知 |
| **不労（Flow）** | 資金フロー監視 | 先回り売り・ファイナンス予見フローの検知 |
| **金田（Trader）** | 売買タイミング | 希薄化リスク「低」確認まで全力ロング待機 |
| **豆田（Risk）** | リスク定量化 | 希薄化シナリオの数値計算・FV下方修正判断 |
| **深井（FairValue）** | 理論株価統合 | 竹村チェックリスト完答・希薄化後FV算出 |
