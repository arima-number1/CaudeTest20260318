# 2. 社長指示重点分析

## ① Sunrise 2027 の義務化インパクト定量化

### (A) GS1 US『Sunrise 2027』の内容：スコープと強制力

**GS1 US『Sunrise 2027』とは何か**

GS1（Global Standards One）は国際バーコード標準化機関であり、GS1 USはその米国法人。『Sunrise 2027』は2027年12月31日を期限として、世界の小売POSシステムが**一次元バーコード（UPC-A・EAN-13 等）だけでなく二次元コード（QR Code・GS1 DataMatrix・DataBar等）を読み取れるよう対応義務化**するガイドラインである。

**義務化のスコープ**

| 項目 | 内容 |
|---|---|
| **対象国・地域** | GS1加盟国（全世界116カ国）。特に米国・EU・日本・豪州が先行 |
| **対象業態** | 小売店舗のPOSシステム全般（スーパー・コンビニ・薬局・百貨店等） |
| **対象品目** | 一般消費財（食品・飲料・日用品・医薬品等）全般 |
| **義務化内容** | POSスキャナーが二次元コードを「読める」ようにすること（=スキャナーのハードウェア・ソフトウェア対応）。かつメーカー側が商品に二次元コードを印刷すること |
| **期限** | 2027年12月31日（Sunrise 2027 = 2027年の夜明け） |

**強制力の評価**

```
【強制力の構造】
GS1ガイドライン → 直接的法的ペナルティは存在しない
　↓
しかし実質的強制力は極めて高い理由：
(1) 小売業者（ウォルマート・Amazon・カルフール等）がサプライヤーに対し
    「2027年末までに二次元コード対応ラベルでなければ棚に入れない」
    という取引条件化を進行中（De facto強制）
(2) GS1 US加盟の小売チェーンはPOS対応義務を負う形で加盟契約に組み込まれつつある
(3) EU・日本では流通業界団体が自主規制として採用済み
```

> **深井評価：** 法律的ペナルティはないが、「取引できなくなる」という実質的強制力はGDPR並みに強い。特にウォルマート・クローガーなど米国大手小売がサプライヤーへの要求として明示化している点が重要。**「任意」ではなく「取引条件」として機能している。**

**GS1の過去の規格移行事例（延期リスク評価用）**

| 規格 | 当初期限 | 実際の定着 | 備考 |
|---|---|---|---|
| EAN-13 国際統一化 | 1977年目標 | 1980年代前半 | 約5〜7年遅延 |
| GS1-128 物流ラベル | 1989年 | 1995年頃 | 約5年遅延 |
| RFID義務化（Sunrise 2005） | 2005年 | 部分対応のみ | 大幅遅延・骨抜き（コスト問題） |
| **Sunrise 2027** | **2027年末** | **現在進行中** | RFID失敗の教訓を踏まえ、スキャナー対応コストが大幅に低い二次元コードを選択 |

> **深井注目点：** 2005年のRFID義務化（Sunrise 2005）は大幅遅延・骨抜きとなった。しかし二次元コード対応は**RFIDと異なりスキャナー1台あたりの追加コストが低く**（ソフトウェアアップデートで対応可能な機種も多い）、移行ハードルが格段に低い。延期リスクは存在するが、RFID失敗の再現確率は低いと判断する。
