## 4. 会社側のAIポジティブ主張の整理

エクスモーションが（公式・非公式に）主張しているAI影響のポジティブ側面を整理する。これらの主張を「真贋検証」のベースラインとする。

### 4.1 主張1：AI複雑化→検証・保証ニーズの拡大

**会社主張の要旨：**
「AIがコード生成・設計支援を担うようになっても、生成されたコードの正確性・安全性を検証する工程は人間が行わなければならない。特に機能安全（ISO 26262、IEC 61508）が要求される組込みソフトウェアでは、AI生成コードの認証要件が既存の手動コードよりもむしろ厳しくなる可能性がある。これはエクスモーションのツール・コンサルティングへの需要増加を意味する。」

**論拠の構造：**
```
AI → コード生成の民主化
     ↓
開発プロジェクト数の増加
     ↓
品質保証・検証プロセスの需要増
     ↓
エクスモーションのツール・コンサル需要増
```

### 4.2 主張2：開発者AI活用→プロジェクト数増加→ライセンス需要増

**会社主張の要旨：**
「GitHub Copilot等のAIコーディングアシスタントにより、組込みソフトウェア開発者の生産性が向上する。これにより、より多くの開発プロジェクトが走るようになり、管理ツール・設計ツールへの需要が増加する。」

**論拠の構造：**
```
AI → 開発者1人当たりの生産性向上
     ↓
同じ人員でより多くのプロジェクト対応可能
     ↓
プロジェクト管理・品質管理ツールの需要増
     ↓
ライセンス収益の拡大
```

### 4.3 主張3：安全クリティカル領域でのAI活用コンサル需要

**会社主張の要旨：**
「顧客企業（自動車OEM・Tier-1等）がAIを安全クリティカルシステムに活用したいが、どのように使えばよいかわからない。エクスモーションはISO 26262・IEC 61508に精通した専門家として、AI活用の安全要件整備コンサルティングを提供できる。」

**論拠の構造：**
```
AI → 安全クリティカル分野での活用ニーズ
     ↓
しかし規制・認証要件が不明確
     ↓
外部専門コンサルタントへの相談需要増
     ↓
エクスモーションへの新規コンサル需要
```

### 4.4 主張の総合評価（深井初期見解）

これら3つの主張は、**いずれも一定の合理性を持つ**。特に主張1（AI検証需要）と主張3（安全コンサル需要）は短期（0〜2年）においては現実として既に顕在化している。1Q FY2025の+54.9%成長はこれらのポジティブ要因を実証しているとも解釈できる。

**しかし**、これらの主張には重大な「見えにくいリスク」が隠れている。以下の批判的検証セクションで、会社側が語らない（あるいは認識していない可能性がある）リスクを定量化する。

## 5. 深井批判的検証：AIリスク3点と定量化

### リスク分析の枠組み

深井は以下の問いを立てる：

> **「会社はAIを追い風と表現しているが、同じAIがエクスモーションの既存ビジネスモデルを侵食するリスクはないか？」**

この問いに対して、3つの具体的なリスクを分析する。

---

### リスク1：AI代替リスク（コンサルティング工数の消失）

#### 1.1 リスクの本質

エクスモーションのコンサルティング事業は「組込みソフトウェア開発プロセスの改善を人月型で提供する」モデルである。問題は、**このコンサルタントが行う作業の一部がAIに代替可能かどうか**である。

#### 1.2 代替可能な作業の特定

| コンサルタント業務 | AI代替可能性 | 代替時期 | 代替確率 |
|----------------|------------|--------|---------|
| AUTOSARコンプライアントコードスタブの生成 | **高** | 1〜3年 | 60〜70% |
| ステートマシン図の作成（要件から自動生成） | **高** | 1〜3年 | 50〜65% |
| テストベクタの自動生成 | **中〜高** | 2〜4年 | 40〜60% |
| コードレビュー（パターン検査） | **中** | 2〜4年 | 30〜50% |
| プロセス改善提言（一般的なベストプラクティス） | **中** | 3〜5年 | 25〜40% |
| 機能安全認証の文書作成支援 | **低〜中** | 3〜6年 | 20〜35% |
| 顧客固有プロセスの設計・カスタマイズ | **低** | 5年超 | 10〜20% |
| 組織変革・文化変革のファシリテーション | **極低** | 10年超 | 5%未満 |

#### 1.3 定量化：コンサルティング収益への影響

```
【前提】
コンサルティング収益：総収益の約40%
AI代替可能な業務比率（3〜5年で）：代替可能業務の平均代替確率
　= (70%+60%+50%+40%+30%) ÷ 5 × コンサル工数の60%
　≒ 50% × 60% = 30%（コンサルタント工数の30%がAI代替リスクにさらされる）

→ AI代替による収益減少（最大ケース、3〜5年）：
  総収益 × 40% × 30% = 総収益の12%が代替リスク

→ 保守的見積もり（実際の代替は段階的かつ完全代替ではない）：
  実際の収益影響 = 上記の50〜60% = 総収益の6〜7%
```

**結論：** コンサルティング収益の最大30%が3〜5年でAI代替リスクにさらされるが、実際の収益影響は総収益の6〜7%程度に留まると推定する。これは看過できないが、ビジネスモデルを破壊するレベルではない。

#### 1.4 「代替の壁」：安全クリティカル認証要件

重要な留意点として、機能安全認証（ISO 26262、IEC 61508）の要件がAI代替の「ブレーキ」として機能する：

- 認証当局（テュフなど）はAI生成成果物の自律的認証を現時点では認めていない
- 人間の専門家によるレビュー・承認が認証条件として明示的に要求されている
- これにより、「完全AI代替」ではなく「AIアシスト＋人間監督」モデルへの移行が現実的なパス

**つまり：** AIはコンサルタントの「仕事を奪う」のではなく、「仕事のやり方を変える」のが短中期の現実である。エクスモーションが自社コンサルタントをAI活用にアップスキリングできれば、1人当たり生産性向上による収益性改善が実現する。

### リスク2：ツールベンダー大手の侵食（プラットフォームリスク）

#### 2.1 リスクの本質

エクスモーションの独自ツール（TERAS、Astah）が存在するニッチに、大手ツールベンダーがAI機能を追加してネイティブに進出してくるリスクである。

#### 2.2 競合ツールベンダーのAI展開状況

**MathWorks（MATLAB/Simulink）**
- 2024年よりSimulink AI Generationの試験提供開始
- 要件文章→Simulinkモデルの自動生成機能を研究段階で開発
- もし実用化されれば、モデリング作業コストが激減→Astahの差別化が薄れる可能性

**Vector Informatik**
- AUTOSAR対応ツール（DaVinci Developer等）にAI支援機能を追加
- コード生成・検証の自動化を進めている
- 自動車Tier-1での利用率が高く、エクスモーションの顧客と重複

**IBM Engineering（旧DOORS/Rational）**
- AI駆動の要件管理（Watsonを活用）
- エンタープライズ向けに大手OEMに浸透

**GitHub（Microsoft）/ JetBrains**
- 組込みC/C++開発のためのAI支援コーディング
- 汎用的だが組込みドメインへの特化が進んでいる

#### 2.3 定量化：ツールライセンス収益への影響

```
【前提】
ツールライセンス収益：総収益の約50%

競合大手AIツール侵食リスク：
- 短期（1〜2年）：影響極小（製品差別化と顧客ロックインが有効）
- 中期（2〜5年）：15〜25%の顧客が大手ツール統合プラットフォームへ移行リスク
- 長期（5年超）：30〜40%の顧客が移行する可能性（最悪ケース）

中期リスク（基本ケース）：
ツール収益の15〜25%が侵食リスク
→ 総収益への影響：50% × 20% = 10%（中期、基本ケース）

但し、移行は段階的・緩やかであり：
実際の年間影響：総収益の2〜3%/年（3〜5年かけて累積10%）
```

**緩和要因：**
1. **スイッチングコスト：** エクスモーションのツールは顧客の開発プロセスに深く統合されており、移行コスト（再教育・プロセス再設計・データ移行）が高い
2. **日本語・日本市場特化：** 大手外資ツールは日本の自動車サプライチェーン固有の慣行に未対応な部分が残る
3. **SMB市場：** 大手ツールの高い価格・複雑性は中小開発組織では採用困難。エクスモーションの主要ターゲットに大手が本格的に踏み込むには時間がかかる

### リスク3：顧客（自動車OEM/Tier-1）のAI内製化

#### 3.1 リスクの本質

トヨタ、デンソー、ボッシュ、コンチネンタルなどの主要自動車メーカー・Tier-1サプライヤーが内部AI開発チームを急速に構築している。これが成熟した場合、エクスモーションのコンサルタントに依頼していた業務を内製化するリスクがある。

#### 3.2 AI内製化の進捗評価（2026年時点）

```
【大手OEM・Tier-1のAI内製化進捗（2026年現時点）】

トヨタ：
- Woven Cityでの自動化・AIインフラ開発
- ソフトウェア定義車両（SDV）向けAI開発チーム構築中
- ただし、組込みソフトウェア安全認証の内製化は5年以上のタイムラグ

デンソー：
- デジタルイノベーション推進室が社内AI活用を推進
- 外部コンサル依存度の段階的低下が予想されるが、急速ではない

ボッシュ、コンチネンタル（日本事業）：
- 本社（欧州）でのAI開発成果を日本に展開
- 機能安全認証領域は欧州基準と日本市場の差があり、エクスモーションの役割残存

```

#### 3.3 定量化：内製化による収益影響


## 6. 「人間の介在価値」が残る領域（非代替性の核心）

批判的検証の結論として、深井はエクスモーションの「AIに代替されない価値」を特定する。

### 6.1 機能安全認証の「人間サイン」要件

**最重要の非代替性：機能安全認証は人間の専門家署名を必須とする**

ISO 26262（自動車機能安全）、IEC 61508（機能安全全般）では：
- 安全要件の妥当性確認（Validation）：認定された人間の専門家が実施・署名
- 安全ケース（Safety Case）の作成・承認：人間の判断と責任が要求される
- FMEA（故障モード影響解析）の承認：エンジニアリング的判断が必要
- ハザード分析・リスク評価（HARA）：ドメイン知識を持つ人間専門家が不可欠

**AIが生成した成果物をAIが認証することは、現在の規制枠組みでは不可能である。**
人間の専門家（FunctionalSafetyEngineer資格保持者等）のレビュー・承認がなければ、製品は認証を取得できない。エクスモーションのコンサルタントはこの「署名する人間」の役割を担える数少ない専門家集団である。

### 6.2 日本固有の開発文化・言語・慣行

**日本の組込みソフトウェア開発現場の固有性：**

- **言語：** 仕様書・設計文書・コードコメントの大半が日本語
- **JASO（自動車技術会標準）・JASPAR（日本の自動車ソフトウェアアーキテクチャ）：** 日本固有の自動車ソフトウェア標準。海外ツールベンダーの対応が不十分
- **「現地現物」文化：** 顧客の開発現場に入り込み、ともに問題を解決する日本型コンサルティング。ブラックボックスのAIシステムでは代替困難
- **暗黙知の移転：** ベテランエンジニアの暗黙知を若手・新規顧客に移転するコンサル機能はAI単独では代替不可能

### 6.3 顧客プロセスへの深い統合（高いスイッチングコスト）

エクスモーションのコンサルタントは、顧客企業の開発プロセス全体に入り込む。これにより：

- 顧客固有の要件定義テンプレート・チェックリストを共同開発
- 顧客の品質ゲート・レビュープロセスを設計
- 過去プロジェクトのデータ・教訓を蓄積・活用

このような「顧客プロセスへの深い統合」は、AIが外から支援するだけでは再現できない。エクスモーションのコンサルタントが持つ顧客固有知識は、単純な工数代替で失われない価値の核心である。

### 6.4 責任・説明責任の所在

**AIは「責任」を取れない。**

製品事故が発生した際、機能安全認証プロセスを遂行した人間・組織が法的・道義的責任を負う。AI生成成果物の責任の所在が不明確なままである現状では、製品安全に関わる重要判断を人間専門家が行う体制は維持される。

エクスモーションはこの「責任ある専門家」の役割を担うことで、AI時代においても不可欠な存在であり続ける。

### 6.5 非代替性スコアカード

| 業務領域 | AIによる代替可能性 | エクスモーションの非代替性 | 備考 |
|---------|----------------|----------------------|------|

| 顧客固有プロセス設計 | 低 | **高** | 暗黙知・文脈依存 |
| 日本語対応・日本市場特化 | 低〜中 | **高** | 言語・文化バリア |

| 標準プロセスのカスタマイズ | 中 | **中** | AI補助で効率化余地あり |
| ドキュメント作成支援 | 中〜高 | **低〜中** | AI代替リスク最大 |
| コードスタブ生成 | 高 | **低** | 最もAI代替されやすい |

## 8. 結論：BUY維持（但し条件付き）と監視ポイント

### 8.1 総合投資判断

**投資判断：BUY維持（条件付き）**

エクスモーション（4394）は、AI影響の批判的検証を行った結果でも、以下の理由からBUY判断を維持する：

**BUY継続の根拠：**

1. **1Q FY2025の実績が示す現実の強さ：** 営業利益+54.9%、営業利益率15.9%は、現時点ではAIが間違いなく「追い風」として機能していることを示す。

2. **機能安全認証という不可侵の堀：** ISO 26262・IEC 61508の認証要件が存在する限り、エクスモーションの専門コンサルタントの役割は法規制レベルで保護されている。

3. **日本固有市場の参入障壁：** 日本語対応、日本自動車サプライチェーン固有の慣行への精通、JASO/JASPAR対応は、外資大手ツールベンダーが即座に代替できない壁である。

4. **加重フェアバリューの現在株価への優位性：** AI影響を最も悲観的に织り込んだコアケース（¥680〜730）でも現在株価と概ね同水準〜若干の上値余地があり、加重期待FV（¥838）では+29%のアップサイドが存在する。

5. **規模・ニッチの防衛力：** 小型ニッチ企業としての俊敏性を活かし、AIを脅威ではなくツール（AIアシスト機能の自社製品組み込み等）として取り込む可能性がある。

**条件付きの理由（リスク認識）：**

- コンサルティング収益の構造的変化リスクは実在する（3〜5年の時間軸）
- 大手ツールベンダーのAI機能強化スピードによっては、予想より早期に競合圧力が高まる可能性
- 顧客の内製化動向は継続モニタリングが必要

2. **大手ツールの日本市場シェア急拡大：** MathWorksやVector等が日本語化・ローカライズ強化で直接競合に踏み込んだ場合

### 8.5 深井の最終コメント

エクスモーション（4394）は、AI時代の「勝ち組」でも「負け組」でもない、**「賢い適応者」になれるかどうかが問われている企業**である。

機能安全という法規制的な堀、日本語・日本市場という文化的な堀、顧客プロセスへの深い統合という関係的な堀——これらの堀は、生成AIが急速に進化する現在でも、3〜5年では容易に乗り越えられない。

一方で、会社がAIを「追い風だけ」と語るならば、それは半分しか正しくない。コンサルティング工数の部分的AI代替、大手ツールのAI機能強化という「向かい風」も同時に存在する。

深井の結論は明快だ：**エクスモーションは今がバイのタイミングである。ただし、目を開けたまま買え。** 次の2年で業績モメンタムを享受し、3〜4年後にビジネスモデルの適応が確認できれば、さらなる長期保有を検討する。適応に失敗した証拠が出たなら、その時点で判断を変える。
